# Evaluation Dashboard

The evaluation dashboard is a web-based replacement for the CLI `make run-eval-*` workflow. It provides a browser UI for selecting missions, running evaluations, and monitoring progress in real time.

---

## Architecture

```
Browser (localhost:3000)         FastAPI (localhost:8100)         Isaac Sim + ROS2
┌──────────────────────┐        ┌────────────────────────┐      ┌──────────────────┐
│  Next.js Frontend    │  HTTP  │  Dashboard Backend     │ ROS2 │  Mission Manager │
│  - Mission table     │◄──────►│  - REST API            │◄────►│  - /start_mission│
│  - Eval controls     │   WS   │  - WebSocket status    │      │  - /skip_mission │
│  - Live status panel │◄──────►│  - ROS2 Bridge         │      │  - /get_mission  │
│                      │        │  - Result writer       │      │    _result       │
└──────────────────────┘        └────────────────────────┘      └──────────────────┘
```

| Component | Stack | Container |
|:----------|:------|:----------|
| Backend | FastAPI + rclpy (ROS2 Jazzy image) | `costnav-dashboard-backend` |
| Frontend | Next.js 14 + Tailwind CSS | `costnav-dashboard-frontend` |

The backend runs inside the ROS2 container and communicates with the Isaac Sim mission manager via ROS2 services and topics. The frontend is a standalone Node.js container.

---

## Setup and Running

### Prerequisites

A running simulation instance is required before starting the dashboard:

```bash
# Start any navigation stack first, e.g.:
make run-nav2
# or
make run-canvas
```

### Start

```bash
make run-dashboard
```

This launches two containers via the `dashboard` Docker Compose profile:

| Service | URL |
|:--------|:----|
| Dashboard UI | [http://localhost:3000](http://localhost:3000) |
| API (Swagger docs) | [http://localhost:8100/docs](http://localhost:8100/docs) |

### Stop

```bash
make stop-dashboard
```

## Usage

### Mission Table

The dashboard loads missions from a JSON file (default: `config/missions/42.json`). Each row shows:

- **Mission number** (1-indexed)
- **Difficulty** (none / easy / hard)
- **Start and goal coordinates**
- **Status** (result after completion)
- **Time and distance** metrics

You can run individual missions by clicking the **Run** button on each row.

### Batch Evaluation

Use the controls at the top of the page:

| Control | Description |
|:--------|:------------|
| **Timeout** | Per-mission timeout in seconds (default: 241s) |
| **Run All** | Execute all missions sequentially |
| **Run Selected** | Execute only checked missions |
| **Skip** | Skip the currently running mission |
| **Stop** | Cancel the evaluation run |

### Live Status

While an evaluation is running, the live status panel shows:

- Overall progress bar (completed / total)
- Current mission metrics: distance to goal, traveled distance, elapsed time, velocity, contacts, impulse, injury cost
- Post-completion summary: success rate, average time, average distance

## Configuration

### Environment Variables

| Variable | Default | Description |
|:---------|:--------|:------------|
| `MISSION_FILE` | `config/missions/42.json` | Path to mission definitions |
| `LOG_DIR` | `logs` | Directory for result output |
| `NEXT_PUBLIC_API_URL` | `http://localhost:8100` | Backend URL (frontend) |

These can be overridden in `.env` or `docker-compose.yml`.

### Mission File Format

Missions are defined as a JSON file with the following structure:

```json
{
  "missions": [
    {
      "id": 0,
      "difficulty": "none",
      "start": { "x": 10.0, "y": 5.0 },
      "goal": { "x": 50.0, "y": 20.0 },
      "waypoints": [[10.0, 5.0], [30.0, 12.0], [50.0, 20.0]]
    }
  ]
}
```

## Result Output

After an evaluation run completes, results are saved to `LOG_DIR` in two formats:

| Format | Filename | Description |
|:-------|:---------|:------------|
| JSON | `dashboard_eval_<timestamp>.json` | Full results with metadata and per-mission details |
| CSV | `dashboard_eval_<timestamp>.csv` | Flat table compatible with `evaluation_summary.csv` |

Results can also be listed and downloaded via the API:

- `GET /api/results` — list available result files
- `GET /api/results/{filename}` — download a specific file

## API Reference

The full interactive API documentation (Swagger UI) is auto-generated by FastAPI and served at `http://localhost:8100/docs` when the backend is running.

### REST Endpoints

| Method | Path | Description |
|:-------|:-----|:------------|
| `GET` | `/api/missions` | List all missions from the configured JSON file |
| `POST` | `/api/missions/{id}/start` | Start a single mission (query param: `timeout`) |
| `POST` | `/api/eval/start` | Start a batch evaluation (body: `EvalRequest`) |
| `POST` | `/api/eval/stop` | Stop the current evaluation run |
| `POST` | `/api/skip` | Skip the currently running mission |
| `GET` | `/api/status` | Get current evaluation and mission status |
| `GET` | `/api/results` | List saved result files |
| `GET` | `/api/results/{filename}` | Download a result file |

### WebSocket

| Path | Description |
|:-----|:------------|
| `ws://localhost:8100/ws/status` | Live status updates (1 Hz JSON frames) |

### Request / Response Models

**`EvalRequest`** (POST `/api/eval/start`):

```json
{
  "mission_ids": [0, 1, 2, 3],
  "timeout": 241.0
}
```

**`EvalStatus`** (GET `/api/status`, WebSocket `/ws/status`):

```json
{
  "running": true,
  "current_mission_id": 2,
  "completed": 1,
  "total": 4,
  "results": [],
  "current_status": {
    "mission_id": 2,
    "mission_number": 3,
    "result": "pending",
    "in_progress": true,
    "distance_to_goal": 15.3,
    "traveled_distance": 42.7,
    "elapsed_time": 58.2,
    "avg_velocity": 0.73,
    "total_contact_count": 0,
    "people_contact_count": 0,
    "total_impulse": 0.0,
    "injury_cost": 0.0
  }
}
```

## ROS2 Interface

The backend communicates with the mission manager through:

### Topics (Published)

| Topic | Type | Description |
|:------|:-----|:------------|
| `/set_mission_index` | `std_msgs/Int32` | Set the active mission (0-indexed) |
| `/set_mission_timeout` | `std_msgs/Float64` | Set per-mission timeout in seconds |

### Services (Called)

| Service | Type | Description |
|:--------|:-----|:------------|
| `/start_mission` | `std_srvs/Trigger` | Start the active mission |
| `/skip_mission` | `std_srvs/Trigger` | Skip the current mission |
| `/get_mission_result` | `std_srvs/Trigger` | Query mission result (JSON in `message` field) |
